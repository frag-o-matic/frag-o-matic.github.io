Notes

Notes. A necessary evil to compensate for limited greymatter that a human has. Even more necessary when one is juggling too many things at a time. A simple paper and pen system is often a good start, as compared to keeping everything locked in the noggin, but in this digital age, one cannot help but wonder if there are better ways to deal with the problem of keeping notes and finding them one the need arises.

Note-taking Applications

There are plenty of note-taking applications, often with fancy interfaces and cloud sync capabilities. The problem with these though, is that they do not lend themselves to use in a locked down environment such as a college/work environment where the machine being used has a specific configuration and the user is not granted super user priviledges or the permission to install additional software. How then can one keep and reliably find notes in such an environment? Using existing tools and some weird conventions of course!

A Convoluted Setup

A simple plain-text based note-taking setup with support for rudimentary tags can be built using tools found in the standard C or C++ programmer's toolset, viz. A text-editor and cscope (alternatively, ctags with certain changes). Here's how yours truly built a similar system for keeping notes while in college (for the various labs)

  • Create an empty directory to hold the notes, this could be in your $HOME dir for general notes or a sub-dir within the project or doc folder of your current workspace for the project/workspace specific stuff
  • Start with an empty file with a meaningful name (or timestamp using touch $(date +"%Y%m%d%H%M), for example. This invocation can be aliased to a convenient bash command like new_note)
  • Add tags as you would declare a global variable in C, eg: const int meta, mid_term_prj, lab02;
  • Create a list of notes to be indexed using ls > cscope.files and run cscope over these with cscope -b -q -k. This will index all the tags (global variable declaration)
  • To find a note based on tags, run cscope in standalone mode with cscope -d (or from within your editor)
  • To find a note based on content. run grep or similar

Limitations

Obviously this is an improvised system and has lots of limitations compared to a real note-taking program, some important ones are:

  • If your notes are in the same dir as your source, the real variables in your code and the pseudo variables used as tags will get all mixed up
  • This "solution" will not scale well
  • No "sync" capabilities
